###工具：JEB/AndroidKill （需配置JAVA环境）

Android killer 是一款可视化的安卓应用逆向工具，集Apk反编译、Apk打包、Apk签名，编码互转，ADB通信（应用安装-卸载-运行-设备文件管理）等特色功能于一身，支持logcat日志输出，语法高亮，基于关键字（支持单行代码或多行代码段）项目内搜索，可自定义外部工具，大大简化了安卓应用/游戏修改过程中各类繁琐工作。

JEB 是一个功能强大的为安全专业人士设计的Android应用程序的反编译工具。用于逆向工程或审计APK文件，可以提高效率减少许多工程师的分析时间。

MuMu模拟器 是一个网易公司开发的安卓模拟器，安装后可在电脑上运行各类安卓游戏与应用，相比于其他模拟器，安装过程极为简单，兼容性较好，不失为一个方便快捷的模拟器。

这是一个让人讨厌的题，因为题目提示需要使用GDB对demo进行动态调试，但是GDB是Linux下的一个软件，且没有图形界面，故为了顺利运行它，我们需要安装在虚拟机下安装一个Linux系统，此处推荐使用Ubuntu，并安装gdb-peda的插件，同时本题需要使用ida进行静态调试进行辅助，当然本题只使用静态调试亦可解决。本文不再赘述Ubuntu与gdb-peda插件的安装，如有需要请参考相关资料。

首先使用gdb与ida打开文件

只需在终端输入gdb 文件目录 即可使用gdb分析文件
使用ida分析静态汇编语句，于是找到一个关键的函数调用

进入该函数，开始分析
我设置了如下断点：

随后按r开始运行程序至第一个断点，这个时候并不会出现有用的结果，于是按n步进
在跳到第二个断点时，寄存器开始大量写入数据

“gmwOSPVkVQk@\QkY[G@kD[CQFRAXk{gk\v\n@”很可能是一个密文，于是拷贝它备用。
继续步进，可以发现gmw逐渐被替换为SYC
这儿很可能使用了异或运算加密，于是找到gmw与SYC对应的ASCII码进行异或运算，发现结果等于一个定值，即0x34
可以肯定此处是将gmwOSPVkVQk@\QkY[G@kD[CQFRAXk{gk\v\n@与0x34进行异或运算得到flag，于是写一个C语言的脚本辅助运算（当然也可以手算），脚本代码如下：

#include<stdio.h>
int main()
{
    int i;
    char kai[43]= {"gmwOSPVkVQk@\\QkY[G@kD[CQFRAXk{gkPQVASSQFI"};
    for(i=0;i<=43;i++)
    {
        kai[i]^=0x34;
        printf("%c",kai[i]);
    }
    return 0;
}

输出结果为：
SYC{gdb_be_the_most_powerful_OS_debugger}444
这个444显然是多余的，于是删去444，得到flag

SYC{gdb_be_the_most_powerful_OS_debugger}

这个题不使用gdb仍可以做，直接用ida打开文件，然后按f5反编译，此时会出现报错

报错原因是堆栈不平衡。
具体解释如下：
一般是程序代码有一些干扰代码，让IDA的反汇编分析出现错误。比如用push + n条指令 + retn来实际跳转，而IDA会以为retn是函数要结束，结果它分析后发现调用栈不平衡，因此就提示sp analysis failed.
所以需要修改堆栈指针sp，找到问题堆栈指针400802，发现其值为-8

使用alt+k修改400801指针sp为0x0

然后保存f5即可正常反编译了，得到以下关键伪c代码
